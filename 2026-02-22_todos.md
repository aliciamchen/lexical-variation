  - For this run, I tried making a new batch with one `refer + separated` game (because it was originally designed to be able to collect two games, although in retrospection this was tight), but i made it after some participants were already assigned to the other batch, and once that game started those participants got kicked. It turns out that Empirica cannot move players between games if they are different treatments. So we should pre-make all the batches, and set `preferUnderassignedGames: true` (more details in the empirica documentation: https://docs.empirica.ly/overview/randomization-and-batches)
